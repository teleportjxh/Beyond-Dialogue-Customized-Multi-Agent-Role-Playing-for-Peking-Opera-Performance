## 任务目标

## 完成内容

### 1. 代码重构
将原有的500行单文件脚本 `extract.py` 重构为模块化架构，创建了以下文件结构：

```
src/
├── config.py                          # 全局配置管理
└── data_extraction/                   # 数据提取模块
    ├── __init__.py                    # 模块初始化
    ├── main.py                        # 主入口文件
    ├── data_models.py                 # 数据模板定义
    ├── llm_client.py                  # LLM客户端封装
    ├── utils.py                       # 工具函数集合
    ├── extractor.py                   # 核心提取器实现
    └── README.md                      # 模块文档
```

### 2. 模块说明

#### config.py (全局配置)
- 集中管理API配置、路径配置、提取参数
- 提供配置验证方法
- 便于后续模块复用配置

#### data_models.py (数据模板)
- 定义5个核心数据模板
- 提供模板深拷贝方法
- 确保数据结构一致性

#### llm_client.py (LLM客户端)
- 封装OpenAI API调用
- 管理不同温度参数的LLM实例
- 统一错误处理

#### utils.py (工具函数)
包含4个工具类：
- **FileManager**: 文件和目录管理
- **CharacterIDManager**: 角色ID管理
- **JSONProcessor**: JSON处理和验证
- **ScriptProcessor**: 剧本处理

#### extractor.py (核心提取器)
包含3个核心类：
- **RoleInfoExtractor**: 角色信息提取
- **DialoguePerformanceExtractor**: 对话和表演提取
- **CharacterDataExtractor**: 主控制器

#### main.py (主入口)
- 提供命令行接口
- 友好的用户交互
- 完整的错误处理

#### README.md (文档)
- 详细的使用说明
- 完整的代码架构说明
- 数据格式文档
- 注意事项和扩展性说明

## 使用方法

### 快速开始

```bash
# 在项目根目录运行
python src/data_extraction/main.py
```

### 程序化使用

```python
from src.data_extraction import CharacterDataExtractor

extractor = CharacterDataExtractor()
extractor.process_all_characters()
```

## 输出结果

程序会生成以下数据：

1. **角色信息**: `character/{角色名}/profile.json`
   - 包含角色的基本信息和各剧本中的角色描述

2. **角色数据**: `character_data/{角色名}/data.json`
   - 包含对话数据和表演数据

3. **错误日志**: `error_json/` (如果有解析失败的JSON)
