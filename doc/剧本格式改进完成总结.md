# 剧本格式改进完成总结

## 📅 完成时间
2025年11月13日

## 🎯 改进目标
解决生成剧本格式混乱的问题，使动作、表情、念白、唱词清晰分离，对齐示例剧本格式。

## 📋 问题分析

### 原始问题
1. **动作和表情混乱**：【念】、【做】标签混用，动作描述和台词混在一起
2. **格式不清晰**：缺少场景设定、出场角色说明
3. **系统信息干扰**：包含不应出现的系统提示
4. **层次结构混乱**：没有清晰的场景-角色-表演流程层次

### 参考示例
参考了 `example/煮酒论英雄_完整剧本(1).txt`，该示例具有：
- 清晰的 Markdown 层次结构（###、####）
- 规范的标签使用（[动作]、[念白]、[唱词]、[提示]）
- 完整的场景设定和出场角色说明
- 内容类型清晰分离

## 🔧 实施方案

### 1. 重写格式化器
**文件**：`src/script_generation/script_formatter.py`

**核心改进**：
- 完全重构 `format_script()` 方法
- 新增 `_organize_by_scenes()` - 按场景组织对话
- 新增 `_parse_content()` - 解析并分离各类内容
- 改进 `_format_dialogue()` - 规范化对话格式
- 改进 `_format_single_scene()` - 添加场景设定和出场角色

### 2. 内容解析逻辑
使用正则表达式分离不同类型的内容：

```python
# 提取锣鼓经（入场动作）
entrance_pattern = r'\[〖[^〗]+〗[^\]]+\]'

# 提取唱词
singing_pattern = r'\(唱\)\s*\*\*\[([^\]]+)\]\*\*\s*((?:[^\n]+\n?)+?)(?=【|$)'

# 提取念白
speech_pattern = r'【念】（韵白）([^\【]+)'

# 提取动作
action_pattern = r'【做】\s*\[([^\]]+)\]'
```

### 3. 格式规范化
**标签映射**：
- 原始【念】→ 新格式 `[念白]`
- 原始【做】→ 新格式 `[动作]`
- 原始(唱) → 新格式 `[唱词]`
- 锣鼓经 → `[动作]`（入场动作）
- 情绪 → `(内心：...)`

## 🎓 技术要点

### 正则表达式技巧
1. **非贪婪匹配**：`(?:[^\n]+\n?)+?` 避免过度匹配
2. **前瞻断言**：`(?=【|$)` 确定唱词结束位置
3. **分组捕获**：提取板式和歌词内容

### 数据处理流程
```
原始对话历史
    ↓
按场景组织（跳过系统消息）
    ↓
解析每条对话内容
    ↓
分离锣鼓经、唱词、念白、动作
    ↓
格式化为 Markdown
    ↓
输出完整剧本
```

### 关键设计决策
1. **保留原始数据**：不修改对话历史JSON，只改格式化输出
2. **向后兼容**：新格式化器可处理旧数据
3. **模块化设计**：各解析函数独立，便于维护
4. **错误容忍**：解析失败时使用默认值，不中断流程
