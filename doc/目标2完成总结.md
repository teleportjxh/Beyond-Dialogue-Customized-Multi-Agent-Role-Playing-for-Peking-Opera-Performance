# 目标2完成总结：RAG检索系统

## 完成时间
2025年11月11日

## 目标概述
构建基于语义相似度的RAG（Retrieval-Augmented Generation）检索系统，用于从京剧剧本数据中检索相关场景片段，为后续的多Agent剧本生成系统提供上下文增强。

## 重要改造说明

### 数据源变更
**原计划**：从`character_data/data.json`提取对话和表演数据  
**实际实现**：直接从`enhanced_script/*.txt`文件提取场景数据

**变更原因**：用户提出"现在情况发生了变化，我们更希望直接从enhanced_script下从txt中提取RAG"，要求改变数据提取方式以获得更完整的剧本上下文。

## 系统架构

### 核心模块
RAG系统位于 `src/rag_system/` 目录，包含以下模块：

1. **vector_processor.py** - 向量化处理器（完全重写）
   - 从enhanced_script目录加载剧本txt文件
   - 按场景分段提取文本内容
   - 调用OpenAI Embedding API进行向量化
   - 批量处理以提高效率（batch_size=50）

2. **vector_store.py** - 向量数据库管理器（更新统计方法）
   - 使用FAISS创建和管理向量索引
   - 支持向量的保存和加载
   - 提供基础搜索、按角色搜索、按类型搜索功能
   - 动态统计所有文档类型

3. **semantic_retriever.py** - 语义检索器（保持不变）
   - 智能提取查询中的角色名称和关键词
   - 支持多角色联合检索
   - 相似度过滤和结果排序
   - 提供场景的专项检索

4. **scene_enhancer.py** - 场景增强器（保持不变）
   - 格式化检索结果为可读文本
   - 生成多Agent系统的上下文提示
   - 为特定角色Agent格式化上下文

5. **main.py** - 系统主入口（保持不变）
   - 命令行接口（CLI）
   - 支持构建索引、搜索、场景增强、交互式搜索

## 主要改动详情

### 1. vector_processor.py 完全重写

**核心变化**:
- 移除了从`character_data/data.json`读取的所有方法
- 新增从`enhanced_script`目录读取txt文件的方法
- 改变数据提取策略：从"对话+表演"改为"场景分段"

**新增方法**:

1. `parse_script_file(file_path)` - 解析单个剧本文件
   - 从文件路径提取角色名
   - 从文件名提取剧本ID和名称（格式：01019004_舌战群儒_enhanced_script.txt）
   
2. `extract_scenes(script_data)` - 提取场景分段
   - 使用正则表达式识别【第X场】标记：`【第[一二三四五六七八九十百]+[场幕]】`
   - 提取剧情大纲（第一个场景之前的内容，长度>50字符）
   - 按场景分段，每个场景作为一个检索单元
   - 如果没有场景标记，将整个剧本作为一个单元
   
3. `process_enhanced_scripts()` - 处理所有剧本文件
   - 遍历enhanced_script下的所有角色目录
   - 处理每个角色的所有剧本文件
   - 返回所有场景的文档列表

**保留方法**:
- `vectorize_documents()` - 批量向量化（逻辑不变）
- `process_all_characters()` - 主入口方法（内部调用改为新方法）

### 2. vector_store.py 统计方法更新

**修改内容**:
- `get_statistics()` 方法改为动态统计所有文档类型
- 不再硬编码只统计dialogue和performance
- 新增对outline、scene、full_script等类型的统计
- 保持向后兼容性（仍返回dialogue_count和performance_count字段，值为0）

### 3. 数据结构变化

**旧格式** (从data.json提取):
```json
{
  "id": "孙悟空_dialogue_0",
  "character": "孙悟空",
  "title": "金钱豹",
  "type": "dialogue",
  "text": "剧本: ...\n角色: ...\n对话内容: ...",
  "metadata": {
    "role_bot": {...},
    "role_user": {...},
    "scene": "",
    "tags": [],
    "relation": ""
  }
}
```

**新格式** (从enhanced_script提取):
```json
{
  "id": "doc_0",
  "character": "孙悟空",
  "title": "金钱豹",
  "script_id": "01013014",
  "type": "scene",
  "text": "【第一场】\n场景设定: ...\n对话内容: ...",
  "metadata": {
    "character_name": "孙悟空",
    "script_id": "01013014",
    "script_name": "金钱豹",
    "scene_number": 1,
    "scene_name": "第一场",
    "type": "scene"
  }
}
```

**文档类型**:
- `outline` - 剧情大纲
- `scene` - 场景片段
- `full_script` - 完整剧本（当没有场景分段时）

## 技术实现

### 向量化技术
- **模型**: OpenAI text-embedding-3-small
- **维度**: 1536
- **批处理**: 每批50个文档，减少API调用
- **数据来源**: enhanced_script目录下的txt文件

### 向量数据库
- **引擎**: FAISS (Facebook AI Similarity Search)
- **索引类型**: IndexFlatL2（L2距离/欧氏距离）
- **存储**: 
  - faiss.index: 向量索引文件
  - documents.json: 文档元数据（不含向量）

### 场景提取规则
1. 使用正则表达式 `【第[一二三四五六七八九十百]+[场幕]】` 识别场景标记
2. 第一个场景之前的内容作为剧情大纲（如果长度>50字符）
3. 每个场景包含从场景标记到下一个场景标记之间的所有内容
4. 如果没有场景标记，整个剧本作为一个文档

## 使用方法

### 1. 构建向量索引
```bash
# 首次构建
python -m src.rag_system.main build

# 重建索引（删除旧索引）
python -m src.rag_system.main build --rebuild
```

### 2. 语义搜索
```bash
# 基础搜索
python -m src.rag_system.main search "诸葛亮舌战群儒" --top-k 5

# 保存结果到文件
python -m src.rag_system.main search "孙悟空大战天兵" --top-k 10 --output results.json
```

### 3. 场景增强
```bash
# 为剧本生成提供增强上下文
python -m src.rag_system.main enhance "诸葛亮和孙悟空煮酒论英雄" --top-k 5
```

### 4. 交互式搜索
```bash
python -m src.rag_system.main interactive
```

## 问题修复记录

### EOF错误修复（2025-11-11）
**问题描述**: 在交互式模式下使用管道输入时，第一次查询执行后会陷入无限循环，不断输出"错误: EOF when reading a line"。

**原因分析**: 当使用管道输入时，输入流在第一次读取后就结束了，但程序继续尝试读取输入，导致EOFError异常被捕获但未正确处理，造成无限循环。

**解决方案**: 在 `interactive_search` 方法中添加 `EOFError` 异常处理：
```python
except EOFError:
    # 输入流结束（如管道输入），正常退出
    print("\n输入结束，退出交互模式")
    break
```

**修复效果**: 
- ✅ 管道输入模式下能正常执行查询并退出
- ✅ 交互式模式下Ctrl+D或输入流结束时正常退出
- ✅ 不再出现无限循环的EOF错误

### Python缓存问题（2025-11-11）
**问题描述**: 代码更新后仍使用旧逻辑，导致功能异常。

**解决方案**: 清除Python缓存：
```bash
find . -type d -name "__pycache__" -exec rm -rf {} +
find . -type f -name "*.pyc" -delete
```

**修复效果**: ✅ 代码更新后立即生效
