# Beyond Dialogue: Customized Multi-Agent Role-Playing for Peking Opera Performance

> 超越对话：面向京剧表演的定制化多智能体角色扮演系统

## 项目概述

本项目构建了一套面向京剧表演的**端到端多智能体剧本生成与视频合成系统**，覆盖从原始剧本数据采集到最终数字京剧视频输出的完整流水线。系统由四大阶段组成：

1. **数据增强阶段**（Workflow 架构）：京剧剧本 PDF → 文本提取 → 视频切片 + 视觉模型增强 → 结构化增强剧本
2. **RAG 检索增强阶段**（三阶段 Pipeline）：多级语义切分 → Query 改写 + Hybrid 混合检索 + Rerank 重排序
3. **多智能体剧本生成阶段**（CrewAI 框架）：编剧/导演/演员/服装设计/场景设计 5 类 Agent 协作创作
4. **视频生成阶段**（导演 Agent 审查）：剧本 → Prompt 提取 → 视频生成 → 导演评估 → 迭代优化

## 各阶段详解

### Stage 1: 数据增强（Workflow 架构）

采用 Workflow 流水线架构，将原始京剧剧本转化为结构化增强剧本：

```
原始数据源                    处理流程                         输出
─────────                    ────────                         ────
京剧剧本 PDF (55部)  ──►  PDF解析/文本提取  ──►  txtdata/ (纯文本)
                              │
京剧表演视频         ──►  视频切片/关键帧提取  ──►  视频帧序列
                              │
                              ▼
                     视觉模型(GPT-4V)语义增强
                     - 补充表演动作细节
                     - 识别服装脸谱特征
                     - 标注唱腔身段信息
                              │
                              ▼
                     enhanced_script/ (55部增强剧本)
                     - 包含唱腔、念白、武打、脸谱、服装等完整描述
```

**数据规模**：

| 角色 | 行当 | 剧本数量 | 代表剧目 |
|------|------|---------|---------|
| 诸葛亮 | 老生 | 21 部 | 空城计、失街亭、三顾茅庐、群英会 |
| 孙悟空 | 武生/猴戏 | 13 部 | 安天会、芭蕉扇、盘丝洞、大闹御马监 |
| 赵匡胤 | 老生 | 22 部 | 斩黄袍、龙虎斗、送京娘、陈桥兵变 |

### Stage 2: RAG 检索增强（三阶段 Pipeline）

基于 FAISS 向量数据库的增强检索系统，为剧本生成提供精准的历史表演知识。

#### 2.1 多级语义切分

将增强剧本按语义结构切分为 5 种文档类型（而非固定长度切分），保持语义完整性：

| 文档类型 | 说明 | 数量 |
|----------|------|------|
| `plot_summary` | 剧情大纲摘要 | 55 |
| `character_profile` | 角色描述（扮相、行当、脸谱） | 217 |
| `dialogue` | 念白对话片段 | 133 |
| `singing` | 唱腔片段（西皮/二黄） | 242 |
| `performance` | 舞台表演动作（武打/身段） | 165 |

- **总文档数**：812 个语义块
- **切分参数**：MIN_CHUNK=150, MAX_CHUNK=1200, OVERLAP=80 字符

#### 2.2 增强检索 Pipeline

```
用户查询 → ① Query改写/扩展 → ② Hybrid检索(Dense+BM25) → ③ Rerank重排序 → Top-K结果
```

| 组件 | 技术 | 作用 |
|------|------|------|
| **Query Rewrite** | 同义词还原 + 意图补全 + 子问题拆解 | 扩大召回覆盖面 |
| **Dense Retrieval** | text-embedding-3-small + FAISS IndexFlatIP | 语义相似度检索 |
| **BM25 Retrieval** | 京剧专有名词分词 + BM25 评分 | 关键词精确匹配 |
| **Hybrid Fusion** | RRF (Reciprocal Rank Fusion) | 融合 Dense + BM25 结果 |
| **Rerank** | 六维度特征加权重排 | 精排 Top-K 提升精度 |

#### 2.3 消融实验结果

在 28 个测试查询上的评估结果：

| 配置 | R@5 | P@3 | NDCG@5 | MRR |
|------|-----|-----|--------|-----|
| A: Baseline (Dense Only) | 0.179 | 0.202 | 0.195 | 0.394 |
| C: + Hybrid (Dense+BM25) | 0.236 | 0.262 | 0.254 | 0.469 |
| D: + Rerank | 0.279 | 0.310 | 0.289 | 0.465 |
| **G: Hybrid + Rerank** | **0.314** | **0.369** | **0.333** | **0.530** |
| H: Full Pipeline (QR+Hybrid+Rerank) | 0.314 | 0.357 | 0.331 | 0.529 |

**关键发现**：Full Pipeline vs Baseline — Recall@5 **+75.7%**，Precision@3 **+76.7%**，MRR **+34.2%**

> 📖 RAG 系统详细技术文档见 [`src/rag_system/README.md`](src/rag_system/README.md)

### Stage 3: 多智能体剧本生成（CrewAI 框架）

采用 **CrewAI** 作为多智能体编排框架，5 类专业 Agent 协作完成 4 阶段创作流程：

#### Agent 角色分工

| Agent | 角色 | 职责 | 可委派 |
|-------|------|------|--------|
| **编剧 Agent** | 京剧编剧 | 创作剧本大纲、审查服装/场景设计、规划演员行动 | ✅ |
| **服装设计 Agent** | 京剧服装设计师 | 设计角色服装、脸谱方案，接受编剧审查 | ❌ |
| **场景设计 Agent** | 京剧场景设计师 | 设计舞台布景、音效方案，接受编剧审查 | ❌ |
| **演员 Agent** | 京剧演员（动态创建） | 扮演特定角色，生成对话，自我审查 | ❌ |
| **导演 Agent** | 京剧导演 | 控制对话流程，审查演员表演，最终评估 | ✅ |

#### 4 Phase 创作流程

```
Phase 1: 大纲创作（编剧 Agent）
  提取角色 → 加载角色档案 → RAG检索历史知识 → 创作剧本大纲

Phase 2: 设计阶段（编剧审查）
  2a. 服装设计 Agent 设计 → 编剧审查 → (修改/通过)
  2b. 场景设计 Agent 设计 → 编剧审查 → (修改/通过)

Phase 3: 对话生成（导演控制 + 演员表演）
  每场戏循环：
    编剧规划行动 → 导演选择说话者 →
    演员生成对话(含自审) → 导演审查 → (修改/通过) →
    更新滑动窗口记忆 → 下一轮

Phase 4: 最终评估（导演 Agent）
  多维度评分 → 生成评估报告
```

#### 记忆系统

| 记忆类型 | 实现方式 | 用途 |
|----------|----------|------|
| **短期记忆** | 滑动窗口（最近 N 轮 + 超窗摘要压缩） | 保持对话连贯性 |
| **长期记忆** | RAG 向量检索（FAISS 索引） | 检索历史京剧知识 |

#### Tool 工具集

| 工具 | 功能 |
|------|------|
| `RAGSearchTool` | 语义检索历史京剧知识 |
| `CharacterSceneRetrieveTool` | 检索角色相关场景片段 |
| `LoadCharacterProfileTool` | 加载角色档案（profile.json） |
| `LoadCharacterDataTool` | 加载角色详细数据（data.json） |
| `ExtractCharactersTool` | 从用户输入中提取角色名 |
| `ParseJSONTool` | 从 LLM 输出中提取 JSON |
| `FormatScriptTool` | 格式化京剧剧本文本 |

### Stage 4: 视频生成（导演 Agent 审查）

接收 Stage 3 生成的剧本、场景设定和装扮设计，通过视频生成模型（Sora）合成京剧表演视频，并由导演 Agent 进行质量审查和迭代优化：

```
输入                          处理流程                        输出
────                          ────────                        ────
剧本+场景+装扮 JSON  ──►  Prompt 提取  ──►  视频生成(Sora-2)
                                                    │
                                              导演Agent评估
                                              (5维度打分)
                                                    │
                                            ≥7分 → approved/
                                            <7分 → 改进建议
                                                    │
                                              重新生成(保留场景装扮)
                                              最多5次迭代
```

**导演评估维度**：装扮准确性 | 动作质量 | 对话表达 | 场景一致性 | 整体印象

## 快速开始

### 环境要求

- Python 3.10+
- CrewAI 框架
- 大语言模型 API（OpenAI / DeepSeek / 其他兼容接口）
- FAISS（向量检索库）

### 安装依赖

```bash
pip install crewai crewai-tools openai faiss-cpu numpy requests
```

### 配置 API

创建 `src/config.py`，填入 LLM API 密钥和接口地址：

```python
class Config:
    API_KEY = "sk-your_api_key_here"
    BASE_URL = "https://api.openai.com/v1/"
    MODEL_NAME = "gpt-4o"
```

### 运行剧本生成

```bash
# 交互式运行
python main.py

# 命令行直接指定需求
python main.py "请创作一出诸葛亮和孙悟空的京剧"
```

### 完整数据管线（从零开始）

```bash
# Step 1: 数据处理（将原始剧本提取为结构化数据）
python scripts/data_processing/extract.py
python scripts/data_processing/process_with_model_batch.py

# Step 2: 构建 RAG 向量索引
python scripts/rebuild_vector_index.py

# Step 3: 生成剧本
python main.py

# Step 4: 生成视频（需配置视频生成 API）
cd vedio_generation && python run_pipeline.py
```

### 运行 RAG 消融实验

```bash
# 自动生成 Ground Truth
python scripts/auto_ground_truth.py

# 运行 8 组消融实验
python scripts/evaluation/run_ablation.py

# 结果保存在 scripts/evaluation/results/ablation_results_latest.json
```

## 技术栈

| 层面 | 技术 |
|------|------|
| **多Agent框架** | CrewAI |
| **大语言模型** | OpenAI GPT-4o / DeepSeek（可配置） |
| **视觉模型** | GPT-4V（数据增强阶段） |
| **向量检索** | FAISS (Facebook AI Similarity Search) |
| **Embedding** | text-embedding-3-small (1536维) |
| **稀疏检索** | BM25 (自实现，京剧专有名词分词) |
| **混合融合** | RRF (Reciprocal Rank Fusion) |
| **记忆系统** | 滑动窗口短期记忆 + RAG 长期记忆 |
| **视频生成** | Sora-2 |
| **视频评估** | Gemini 2.5 Pro（导演Agent） |
