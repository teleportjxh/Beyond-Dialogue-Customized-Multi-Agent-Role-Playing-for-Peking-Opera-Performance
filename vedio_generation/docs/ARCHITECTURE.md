## 系统概述

### 设计目标

智能视频生成系统旨在实现：

1. **质量保证**：通过导演Agent机制确保每个生成的视频达到质量标准
2. **智能重试**：自动识别问题并重试，提高成功率
3. **模块化**：清晰的模块划分，便于维护和扩展
4. **可配置**：灵活的配置系统，适应不同需求
5. **可追溯**：完整的日志和评估报告，便于问题定位

### 核心特性

- **导演Agent评估**：使用Gemini-2.5-pro进行多维度质量评估
- **优化的Prompt生成**：强调京剧特色和直接表演
- **智能流水线**：逐场景生成，即时评估，通过才继续
- **重试机制**：最多2次重试机会，提供改进建议
- **可选合并**：灵活的视频合并功能

### 模块职责

#### 1. 配置层 (Configuration Layer)

**位置**：`config/`

**职责**：
- 集中管理所有配置参数
- 提供API密钥读取
- 定义目录结构
- 设置评估阈值和权重

**核心类**：
- `Settings`：配置管理类

#### 2. 流水线层 (Pipeline Layer)

**位置**：`src/pipeline/`

**职责**：
- 协调各模块工作
- 实现智能生成流程
- 管理重试逻辑
- 生成流水线报告

**核心类**：
- `IntelligentPipeline`：智能流水线

#### 3. 生成器层 (Generator Layer)

**位置**：`src/generator/`

**职责**：
- 构建优化的prompt
- 调用Sora2 API生成视频
- 管理视频生成任务

**核心类**：
- `PromptBuilder`：Prompt构建器
- `SoraGenerator`：Sora2生成器

#### 4. 评估器层 (Evaluator Layer)

**位置**：`src/evaluator/`

**职责**：
- 评估视频质量
- 生成评估报告
- 提供改进建议
- 判断是否通过

**核心类**：
- `DirectorAgent`：导演Agent评估器

#### 5. 处理器层 (Processor Layer)

**位置**：`src/processor/`

**职责**：
- 视频编辑操作
- 视频合并功能
- 视频信息获取

**核心类**：
- `VideoEditor`：视频编辑器
- `VideoMerger`：视频合并器

#### 6. 工具层 (Utility Layer)

**位置**：`src/utils/`

**职责**：
- 日志记录
- 文件操作
- 通用工具函数

**核心模块**：
- `logger`：日志工具
- `file_manager`：文件管理

## 核心机制

### 1. 导演Agent评估机制

#### 评估流程

```
视频文件
    │
    ▼
视频编码为Base64
    │
    ▼
构建评估Prompt
    │
    ▼
调用Gemini API
    │
    ▼
解析评估结果
    │
    ├─► 音乐质量 (30%)
    ├─► 内容一致性 (30%)
    ├─► 京剧特色 (25%)
    └─► 技术质量 (15%)
    │
    ▼
计算总分
    │
    ▼
判断是否通过 (≥7.0)
    │
    ├─► 通过：移至approved/
    └─► 未通过：移至rejected/，提供改进建议
```

#### 评估维度

1. **音乐质量 (30%)**
   - 京剧音乐的准确性
   - 伴奏的协调性
   - 节奏的把握

2. **内容一致性 (30%)**
   - 与剧本的一致性
   - 角色表演的准确性
   - 场景的完整性

3. **京剧特色 (25%)**
   - 行当表现
   - 脸谱妆容
   - 服装道具
   - 表演风格

4. **技术质量 (15%)**
   - 画面清晰度
   - 色彩还原
   - 音画同步

### 2. 智能重试机制

#### 重试流程

```
生成视频
    │
    ▼
导演评估
    │
    ├─► 通过 (≥7.0) ──► 继续下一场景
    │
    └─► 未通过 (<7.0)
            │
            ▼
        检查重试次数
            │
            ├─► 未达上限 (< MAX_RETRIES)
            │       │
            │       ▼
            │   查看改进建议
            │       │
            │       ▼
            │   重新生成视频
            │       │
            │       └─► 返回评估
            │
            └─► 达到上限
                    │
                    ▼
                标记为失败
                    │
                    ▼
                保存到rejected/
```

#### 重试策略

- **最大重试次数**：2次（可配置）
- **改进建议**：每次失败提供具体建议
- **场景延迟**：重试前等待5秒（可配置）
- **失败处理**：达到上限后保存失败记录

### 3. Prompt优化机制

#### Prompt结构

```
1. 核心约束
   ├─ 直接唱戏，不出现其他内容
   ├─ 严格按照台词表演
   ├─ 突出京剧特色
   └─ 基于对话历史指导

2. 剧本和场景信息
   ├─ 场景描述
   └─ 基础prompt

3. 角色装扮
   ├─ 行当（老生、青衣等）
   ├─ 脸谱（素面、彩绘等）
   ├─ 妆容（传统京剧妆）
   ├─ 服装（蟒袍、官衣等）
   ├─ 配饰（头饰、腰带等）
   └─ 风格（写实、传统等）

4. 表演内容
   ├─ 动作描述
   └─ 唱词内容

5. 音乐伴奏
   ├─ 京剧音乐
   ├─ 传统乐器
   └─ 节奏要求

6. 重要提示
   └─ 只展示表演本身
```

#### 优化要点

- **明确性**：清晰描述每个要素
- **完整性**：包含所有必要信息
- **一致性**：与对话历史保持一致
- **特色性**：强调京剧独特元素

## 工作流程

### 完整生成流程

```
开始
  │
  ▼
加载任务数据
  ├─ prompts.json
  └─ 对话历史.json
  │
  ▼
初始化流水线
  ├─ SoraGenerator
  ├─ DirectorAgent
  ├─ PromptBuilder
  └─ VideoMerger
  │
  ▼
逐场景处理
  │
  ├─► 场景1
  │     │
  │     ├─ 构建Prompt
  │     ├─ 生成视频
  │     ├─ 导演评估
  │     ├─ 判断通过
  │     │   ├─ 通过 ──► 保存到approved/
  │     │   └─ 未通过 ──► 重试或保存到rejected/
  │     │
  │     └─ 延迟5秒
  │
  ├─► 场景2
  │     └─ （同上）
  │
  └─► 场景N
        └─ （同上）
  │
  ▼
可选：合并视频
  │
  ├─ 收集approved/中的视频
  ├─ 按场景顺序排列
  ├─ 合并为完整视频
  └─ 保存到merged/
  │
  ▼
生成流水线报告
  │
  ├─ 总场景数
  ├─ 通过数量
  ├─ 未通过数量
  ├─ 各场景详情
  └─ 合并视频路径
  │
  ▼
结束
```

### 单场景生成流程

```
场景N
  │
  ▼
读取场景数据
  ├─ Prompt信息
  ├─ 对话历史
  └─ 角色装扮
  │
  ▼
构建完整Prompt
  │
  ▼
调用Sora2 API
  │
  ├─ 创建任务
  ├─ 轮询状态
  └─ 下载视频
  │
  ▼
导演Agent评估
  │
  ├─ 编码视频
  ├─ 调用Gemini API
  ├─ 解析评估结果
  └─ 计算总分
  │
  ▼
判断是否通过
  │
  ├─► 通过 (≥7.0)
  │     │
  │     ├─ 移至approved/
  │     ├─ 保存评估报告
  │     └─ 返回成功
  │
  └─► 未通过 (<7.0)
        │
        ├─ 检查重试次数
        │
        ├─► 可重试
        │     │
        │     ├─ 记录改进建议
        │     └─ 重新生成
        │
        └─► 不可重试
              │
              ├─ 移至rejected/
              ├─ 保存评估报告
              └─ 返回失败
```

## 数据流

```
data/
├── prompts/
│   └── {任务名}_prompts.json
│       ├── 场景1
│       │   ├── prompt: 场景描述
│       │   └── character_design
│       │       └── {角色名}
│       │           ├── 行当
│       │           ├── 脸谱
│       │           ├── 妆容
│       │           ├── 服装
│       │           ├── 配饰
│       │           └── 风格

    └── {任务名}_对话历史.json

```

└── API响应数据
```

```
results/
