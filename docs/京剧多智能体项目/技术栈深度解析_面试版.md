## 1. FAISS 向量检索

### 1.1 FAISS 基本原理

FAISS（Facebook AI Similarity Search）是高效的向量相似度搜索库，核心是在高维空间中快速找到最近邻向量。

**IndexFlatL2**（本项目使用）：

```python
import faiss
import numpy as np

# 构建索引（dim=1536 是 text-embedding-ada-002 的维度）
dim = 1536
index = faiss.IndexFlatL2(dim)

# 添加向量（必须是 float32）
vectors = np.array(embeddings, dtype=np.float32)
index.add(vectors)

# 检索：找最相近的 top_k 个
query = np.array([query_vec], dtype=np.float32)
distances, indices = index.search(query, top_k)
# distances: shape (1, top_k)，L2距离
# indices: shape (1, top_k)，对应原始文档的下标
```

**L2 距离 vs 余弦相似度**：
- L2 距离（欧氏距离）：`||a - b||²`，值越小越相似
- 余弦相似度：`a·b / (||a|| ||b||)`，值越大越相似
- 对于 L2 归一化后的向量，L2 距离与余弦距离等价
- text-embedding-ada-002 输出的向量已经归一化，两种方法效果相同

### 1.2 索引持久化与加载

```python
# 保存索引
faiss.write_index(index, "vector_index/faiss.index")

# 加载索引（应用启动时）
index = faiss.read_index("vector_index/faiss.index")
```

**为什么要分开存 `faiss.index` 和 `documents.json`**：

FAISS 只存向量数值，不存原始文本。需要用 documents.json 映射"第 i 个向量对应哪段文本、属于哪个角色"：

```python
# 检索后，通过 indices 反查原始文档
for idx in indices[0]:
    doc = documents[idx]   # {"text": "...", "character": "诸葛亮", ...}
```

### 1.3 元数据过滤（Post-filtering）

FAISS 不原生支持元数据过滤（如"只搜索诸葛亮的片段"），需要手动实现：

```python
def retrieve_by_character(query_vec, character, top_k=5):
    # 多搜一些，再过滤
    distances, indices = index.search(query_vec, top_k * 5)
    
    results = []
    for dist, idx in zip(distances[0], indices[0]):
        doc = documents[idx]
        if doc['character'] == character:   # 角色过滤
            results.append((dist, doc))
        if len(results) >= top_k:
            break
    return results
```

**trade-off**：top_k * 5 的超采样策略简单有效，但如果某角色的剧本数量很少，可能需要更大的倍数。

## 2. OpenAI API 调用与 Prompt 工程

### 2.1 Chat Completions API

```python
from openai import OpenAI

client = OpenAI(
    api_key=Config.API_KEY,
    base_url=Config.BASE_URL   # 支持兼容 OpenAI 的第三方接口
)

def call_llm(system_prompt: str, user_msg: str, temperature: float = 0.7) -> str:
    response = client.chat.completions.create(
        model=Config.MODEL_NAME,   # "gpt-4o"
        messages=[
            {"role": "system", "content": system_prompt},
            {"role": "user",   "content": user_msg}
        ],
        temperature=temperature,
        max_tokens=Config.MAX_TOKENS   # 5000
    )
    return response.choices[0].message.content
```

**temperature 参数的选择**：

| 场景 | temperature | 原因 |
|------|-------------|------|
| 数据提取（extractor.py） | 0.1 | 需要严格按格式输出，不需要创意 |
| 质量评估（judge） | 0.0 | 评分需要确定性，同样输入应得同分 |
| 剧本生成（actor_agent） | 0.7 | 需要一定创意性，但不能太随机 |

### 2.2 JSON 输出的健壮解析

LLM 经常在 JSON 前后多输出解释文字，需要健壮解析：

```python
import json, re

def parse_llm_json(output: str) -> dict:
    # 方法1：直接解析
    try:
        return json.loads(output.strip())
    except json.JSONDecodeError:
        pass
    
    # 方法2：提取 ```json ... ``` 代码块
    match = re.search(r'```json\s*(.*?)\s*```', output, re.DOTALL)
    if match:
        try:
            return json.loads(match.group(1))
        except json.JSONDecodeError:
            pass
    
    # 方法3：提取第一个 { ... } 块
    match = re.search(r'\{.*\}', output, re.DOTALL)
    if match:
        try:
            return json.loads(match.group())
        except json.JSONDecodeError:
            pass
    
    raise ValueError(f"无法解析 LLM 输出为 JSON: {output[:200]}")
```

### 3.1 PDF 文本提取

```python
# utils.py
import pdfplumber  # 或 pypdf2

def read_pdf(file_path: str) -> str:
    text_parts = []
    with pdfplumber.open(file_path) as pdf:
        for page in pdf.pages:
            text = page.extract_text()
            if text:
                text_parts.append(text)
    return "\n".join(text_parts)
```

**京剧剧本 PDF 的特殊处理**：
- 含有大量特殊排版（双栏、竖排转横排）
- 唱词和念白没有统一标记，需要 LLM 辅助识别
- 部分 PDF 是扫描件，需要 OCR（本项目假设已是文字 PDF）

### 3.2 文本切片策略

```python
def split_into_chunks(text: str, chunk_size: int = 512, overlap: int = 64) -> list:
    """
    按句子边界切片，避免在句子中间截断
    overlap=64 保证相邻 chunk 有语义连续性
    """
    sentences = split_by_sentence(text)  # 按句号/问号/感叹号分句
    
    chunks = []
    current_chunk = []
    current_len = 0
    
    for sentence in sentences:
        if current_len + len(sentence) > chunk_size and current_chunk:
            # 当前 chunk 已满，保存并开始新 chunk
            chunks.append("".join(current_chunk))
            # 保留最后 overlap 个字符作为重叠
            overlap_text = "".join(current_chunk)[-overlap:]
            current_chunk = [overlap_text, sentence]
            current_len = len(overlap_text) + len(sentence)
        else:
            current_chunk.append(sentence)
            current_len += len(sentence)
    
    if current_chunk:
        chunks.append("".join(current_chunk))
    
    return chunks
```

**为什么要 overlap**：
防止一个语义完整的唱段被切割成两半，每个 chunk 都包含一点前一段的内容，确保语义完整性。

class AgentBase:
    """所有 Agent 的基类，定义统一接口"""
    
    def __init__(self, profile: dict, retriever=None):

        self.retriever = retriever

    def build_system_prompt(self) -> str:
        raise NotImplementedError
    
    def run(self, context: dict) -> dict:
        raise NotImplementedError
    
    def _chat(self, user_msg: str, use_history: bool = False) -> str:
        """统一的 LLM 调用接口"""
        system = self.build_system_prompt()
        
        if use_history:
            # 多轮对话模式（用于需要迭代修改的场景）
            messages = [{"role": "system", "content": system}]
            messages.extend(self.conversation_history)
            messages.append({"role": "user", "content": user_msg})
            response = llm_client.chat_with_history(messages)
            self.conversation_history.append({"role": "user", "content": user_msg})
            self.conversation_history.append({"role": "assistant", "content": response})
        else:
            response = llm_client.chat(system, user_msg)
        
        return response

### 4.2 DialogueManager 协调流程

```python
# dialogue_manager.py
class DialogueManager:
    """管理多 Agent 的顺序调用和上下文传递"""
    
    def __init__(self, characters: list, retriever: SemanticRetriever):
        self.session_context = {}  # 共享上下文
        self.agents = self._init_agents(characters, retriever)
    
    def run_pipeline(self, play_title: str, character_list: list) -> dict:
        """按顺序运行五个 Agent"""
        
        # Step 1: 导演
        director_output = self.agents['director'].run({
            "play_title": play_title,
            "characters": character_list
        })
        self.session_context['director'] = director_output
        
        # Step 2: 编剧（依赖导演输出）
        screenwriter_output = self.agents['screenwriter'].run(
            self.session_context
        )
        self.session_context['screenwriter'] = screenwriter_output
        
        # Step 3: 演员（每个角色并行？串行？）
        # 这里选择串行，确保角色之间可以参考彼此的输出
        actor_outputs = {}
        for char in character_list:
            actor_output = self.agents[f'actor_{char}'].run(
                self.session_context
            )
            actor_outputs[char] = actor_output
            self.session_context[f'actor_{char}'] = actor_output
        self.session_context['actors'] = actor_outputs
        
        # Step 4 & 5: 场景和服装（可并行，相互独立）
        scene_output = self.agents['scene_setting'].run(self.session_context)
        costume_output = self.agents['costume_designer'].run(self.session_context)
        
        return {
            "outline": screenwriter_output,
            "script": self._merge_scripts(actor_outputs),
            "scene_setting": scene_output,
            "costume_design": costume_output
        }
```

## 6. 向量化与 Embedding

### 6.1 text-embedding-ada-002

本项目使用 OpenAI 的 text-embedding-ada-002 模型进行文本向量化：

```python
def embed(text: str) -> list:
    """将文本转换为 1536 维向量"""
    response = client.embeddings.create(
        model="text-embedding-ada-002",
        input=text
    )
    return response.data[0].embedding
```

**为什么选 ada-002**：
- 1536 维，足够捕捉语义细节
- 成本低（约 $0.0001/1K tokens）
- 对中文支持良好

### 6.2 批量向量化的效率优化

单条调用 vs 批量调用：

```python
# 低效：循环单条调用
for text in texts:
    vec = embed(text)   # 每次 HTTP 请求

# 高效：批量调用（API 支持一次传多条）
def embed_batch(texts: list, batch_size: int = 100) -> list:
    all_embeddings = []
    for i in range(0, len(texts), batch_size):
        batch = texts[i:i+batch_size]
        response = client.embeddings.create(
            model="text-embedding-ada-002",
            input=batch   # 一次传 100 条
        )
        all_embeddings.extend([d.embedding for d in response.data])
    return all_embeddings
```
